DOM笔记1：DOM基础概念、操作

发表于 2017-07-12

DOM:Document Object Model（文档对象模型）

1、元素.childNodes：只读属性 子节点列表集合

标准下：包含了文本和元素类型的节点，也会包含非法嵌套的子节点

非标准下（IE）：只包含元素类型节点，ie7以下不会包含非法嵌套的子节点

childNodes只包含一级子节点，不包含后辈孙级节点

DOM节点的类型：12种（元素节点：1 属性节点：2 文本节点：3）

元素。nodeType：只读属性 当前元素节点类型

获取属性：元素.attributes：只读 属性 属性列表集合

获取属性名称/值：元素.attributes.name[value]

2、元素.children ：只读属性 子节点列表(常用)

标准下：只包含元素类型节点

非标准下：只包含元素类型节点

​ 1.firstChild：第一个子节点

标准下：firstChild会包含文本类型的节点

非标准下：只包含元素节点

​ 元素.firstElementChild：只读 属性 标准下获取第一个元素类型的子节点

​ 一般我们用children[0]来选择第一个节点

3、元素.parentNode: 只读属性 当前节点的父级节点

4、元素.offsetParent:只读 属性 [离当前元素最近的一个有定位]父节点

​ 问题：如果没有定位父级，默认是body[IE7以下，如果当前元素没有定位，则是body。如果有定位，则是html。IE7以下，如果当前元素的某个父级触发了layout，offsetParent就会被指向到这个触发了layout特性的父节点上]

​

5、元素.offsetLeft[Top]：只读属性 当前元素到定位父级的距离（偏移值）

到当前元素的offsetParent的距离

6、

style.width : 样式宽

clientWidth : 可视区宽 （样式宽 + padding）

offsetWidth : 占位宽（样式宽 + padding + border）

7、封装一个获取元素与body距离的函数

8、操作元素属性的多种方式

元素.getAttribute(属性名称); 方法 获取指定元素的指定属性的值

元素.setAttribute(属性名称，属性值); 方法 给指定元素指定的属性设置值

元素.removeAttribute(属性名称); 方法 移除指定的元素的指定的属性

9、元素的创建操作

1、document.createElement(标签名称); 创建元素

2、父级.removeChild(要删除的元素); 删除元素

3、添加到页面中

10、元素的替换

父级.replaceChild(新节点，被替换节点) 替换子节点

appendChild, insertBefore, replaceChild都可以操作动态创建出来的节点，也可以操作已有节点
